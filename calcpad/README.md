# Calcpad

numbr 風格的「筆記型計算機」:大輸入區逐行輸入有意義的資訊,每行右側即時給出答案。內容會即時編進網址,複製網址給別人就能看到一模一樣的內容。純前端、零第三方相依、單頁應用。

---

## 功能目標

使用者輸入「有意義的資訊」就能做計算,而且每種計算功能都**模組化、好加好拆**。

每行獨立計算,互不影響。

### 功能 1:數字計算

```
1 + 2 * (5 - 1)        → 9
1,000 + 2,000,000      → 2001000
```

- 支援 `+ - * /`、括號、運算優先序。
- **千分位逗號**:數字可帶千分位,但分組必須合法 —— 首組 1~3 位,之後每組剛好 3 位(`1,212`、`12,344,444` ✓;`12,12` ✗)。分組不合法的逗號不會被默默吃掉,該行會顯示「看不懂的符號」而非算出錯誤答案。顯示答案時不加逗號(維持純數字)。
- **不使用 `eval()`**:自寫安全的運算式解析(shunting-yard / 簡易 parser),不執行任何使用者輸入的程式碼。

### 功能 2:國際日期計算

```
2026-06-15T20:36:55-07:00 + 2h + 30m + 30s     → 加上 2 小時 30 分 30 秒後的時間
2026-06-15T20:36:55-07:00 to Asia/Taipei       → 轉成 +8 (Asia/Taipei) 的時間
2026-03-01 to 2026-01-01                        → 59 天(兩個時間的期間)
```

- 輸入為 ISO 8601(含時區 offset)。
- 加減運算單位:`d`(日)、`h`(時)、`m`(分)、`s`(秒)。可串接多個(`+ 2h + 30m`),結果沿用原本的時區 offset 顯示。
- 時區轉換:`<時間> to <IANA 時區名>`,用瀏覽器原生 `Intl` API 換算(含夏令時間 DST,零相依)。
- **日期相減算期間**:`<時間> to <時間>`,`to` 後面若是日期就算兩者的期間(天/時/分/秒,取絕對值),否則才當時區名換算。期間以絕對瞬間計算,故兩邊各自的時區 offset 都會被正確納入。
- 認領條件:行首是日期,且後面接「加減運算」或「`to` 時區/日期」其一;光是裸日期(無運算)不認領。

### 註解 / 標題行

```
# 預算(這行不計算)        → 當註解,右側淡色標出
100 + 200                  → 300
```

- **`#` 開頭的行一律不計算**,當「註解 / 標題」:可放段落標題、分隔線,或刻意讓某行不要被算。
- 在核心層處理(與「空行」同層級,不佔用 evaluator),所以不會跟任何計算功能搶。
- 純文字行(無模組認領)本來就不顯示答案;`#` 的差別是**連看起來像算式的內容也強制當註解**(例如 `# 1 + 1` 不會算成 2)。

### 混用偵測

兩個功能**彼此獨立、不會合併使用**。同一行若有兩種以上功能同時宣稱能處理,該行不計算,改在右側顯示提示(例:`不能混用 (math / datetime)`)。

## 網址同步(招牌功能)

整份輸入內容會即時編碼進網址的 `location.hash`,所以:

- 任何人複製這個網址,因為網址後面帶了資料,打開就能看到**相同的內容**。
- 純前端就能做,不需後端;`hash` 不會送到伺服器,長度限制也較寬鬆。

編碼方式:**Base64(URL-safe)**。

- 相對「明文 `encodeURIComponent`」:網址看不出內容,但中文不會爆出大量 `%xx`、較短。
- 不採用「壓縮 + Base64」:計算機內容通常很短,壓縮的固定標頭開銷反而讓短內容變更長,又得改成非同步、相容性較窄,屬於過度設計(YAGNI)。

編解碼獨立成 `js/urlsync.js`(介面 `encode(text)` / `decode(str)`)。日後若真要改成壓縮版,**只改這一個檔案**,核心不動。

## 架構

純前端 ES modules,核心與「計算功能」分離,功能可插拔(沿用本 repo `scrapbook` 的登記表模式)。

```
calcpad/
├── index.html              版面 + 載入 js/main.js
├── styles.css              版面、等寬字體、逐行答案對齊樣式
└── js/
    ├── main.js             控制器:逐行讀取、派發到 evaluator、render 答案、URL 同步
    ├── registry.js         evaluator 登記表(可插拔的關鍵)
    ├── urlsync.js          URL hash ⇄ 文字 的 Base64 編解碼(可單獨替換)
    ├── util.js             共用工具
    └── evaluators/
        ├── math.js         功能 1:安全運算式求值
        └── datetime.js     功能 2:國際日期 / 時區計算
```

### evaluator 登記表(可插拔的關鍵)

`main.js` 不寫死任何計算功能,改用登記表派發每一行的處理:

```js
registerEvaluator({
  name: 'math',
  match:    (line) => boolean,   // 這個模組宣稱能處理這一行嗎?
  evaluate: (line) => string,    // 算出答案文字(或丟出錯誤訊息)
});
```

**逐行派發流程**(每行獨立):

1. 去掉空白;空行則不顯示答案。
2. 讓所有 evaluator 跑 `match(line)`,收集命中的模組。
3. 命中 0 個 → 不顯示答案。
4. 命中 **大於 1 個** → 該行顯示「不能混用」提示(列出衝突的模組名)。
5. 剛好命中 1 個 → 呼叫 `evaluate(line)`,在右側顯示答案;丟錯則顯示錯誤訊息。

**新增一種計算功能** = 寫一個 `evaluators/xxx.js`,在裡面 `registerEvaluator(...)`,並在 `main.js` 加一行載入。
**移除一種功能** = 刪掉該模組 + 拿掉那一行。核心不受影響。

## 安全性

- 數字計算**不使用 `eval()`**:自寫運算式解析,只做數學求值,不執行任何使用者輸入的程式碼。
- 顯示答案時對輸出做跳脫(escape),不插入未經處理的 HTML。

---

## 已知取捨

- 時區換算依賴瀏覽器原生 `Intl`,DST 規則以瀏覽器內建的 IANA 資料為準;時區 offset 顯示用 `longOffset`(較新的瀏覽器才支援)。
- Base64 網址看不出內容(非明文);若需肉眼 debug,可暫時改用明文編碼(只改 `urlsync.js`)。
- 兩個功能互斥不混用;同行多個模組同時認領,以「不能混用」提示取代計算。
- 裸日期(如 `2026-06-15`,無運算)會被數字計算當成減法(= 2005),因為它本身就是合法的算術式;datetime 只在有運算時才認領。需要日期請帶上運算(`2026-06-15 + 0d`)。
- 千分位逗號分組合法性由 number token 的文法把關(`\d{1,3}(,\d{3})+`),不是先刪逗號再算 —— 故分組不對的輸入(`12,12`)會誠實報錯,而非被當成 `1212` 算出來。
- 輸入框用 `wrap=off`(不自動換行),確保右側答案能逐行精準對齊;過長的行改成水平捲動。

## 本機開發

需用本機 HTTP server(ES modules 不能用 `file://` 開):

```bash
cd calcpad
python3 -m http.server 8009
# 開 http://127.0.0.1:8009/
```

## 部署

本 repo 用 GitHub Pages 發佈。app 全用相對路徑,放在子路徑(如 `/calcpad/`)都能運作,不需建置步驟。
