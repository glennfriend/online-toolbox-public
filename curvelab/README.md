# Curvelab

互動式「公式圖形庫」:選一個公式,拉動參數,即時看圖形怎麼變,並解釋每個公式的意義與用途。每種圖都**模組化、好加好拆**。純前端、零第三方相依、單一 HTML 檔 —— 雙擊就能離線開,不需要 server。

---

## 功能目標

把「有意義的公式」做成可以動手玩的圖,讓使用者透過調整參數理解公式,而不只是看靜態圖。

每個圖獨立成一個模組,互不影響;新增、移除一個圖都不會動到其他圖。

---

## 目前的圖

### 圖 1:圓 `x² + y² = r²`

```
拖動圓上的點 → 看 x、y 座標怎麼變
滑桿調 r     → 整個圓變大變小
```

- `x`、`y` 是圓上某個點的座標(可拖,不是參數);`r` 是半徑(用滑桿調)。
- 即時顯示不變式:不管點拖到哪,`x² + y²` 永遠等於 `r²`。
- 半徑滑桿用**整數步進**,讓 `r²` 維持乾淨數字、方便心算與看出 3-4-5 之類的關係。

### 圖 2:拋物線 `y = x²`(曲線生成)

```
按播放或拖 x 滑桿 → 追蹤點沿 x 掃過去,沿路把曲線「畫」出來
```

- 演示「一條曲線 = 點的集合」:給一個 `x`(直虛線),公式算出 `y`(橫虛線),得到一個點;每個 x 算出的點連起來就是曲線。
- x 滑桿用**細步進(0.1)**,因為這裡的重點是「感受連續地掃過去」—— 與圖 1 的整數步進剛好相反,步進大小依用途決定。

### 圖 3:方程式的解 `f(x) = 0`(數線上的點)

```
x^2 - 5x + 6   → 解 x = 2, 3
x^3 - x        → 解 x = -1, 0, 1
sin(x)         → 解 x = 0, ±π, ±2π …
```

- 使用者可**自由輸入只含 x 的式子**;以數值法解出 `f(x)=0` 的根,標在 x 軸(數線)上。
- 「只看數線」切換鈕:藏起曲線只留 x 軸,直接演示「平面曲線 → 數線上的點」的降維關係(把圖 2、圖 3 串起來)。
- **不使用 `eval()`**:自寫安全的運算式解析,不執行任何使用者輸入的程式碼。

## 架構

單一 `index.html`(HTML + CSS + 原生 JavaScript),刻意不用框架、不用 ES modules,核心與「圖形功能」分離,圖可插拔。

```
curvelab/
├── index.html     整個 app:版面 + 樣式 + 程式 + 三個圖模組
└── README.md      本文件
```

`index.html` 內部分四層,各司其職:

```
createPlot()   共用繪圖工具:座標換算、格線、座標軸、把函數取樣成曲線
UI             版面小工具:方程式標題、滑桿列、說明卡、按鈕
圖形模組         registerGraph(...) 登記表(可插拔的關鍵),每個圖一個獨立物件
App            外殼:生成底部切換按鈕、掛載目前的圖、切換時收掉上一個
```

### 圖形登記表(可插拔的關鍵)

`App` 不寫死任何圖,改用登記表管理:

```js
registerGraph({
  id:   'circle',          // 唯一識別
  name: '圓',              // 按鈕顯示名稱
  eq:   'x²+y²=r²',        // 按鈕副標(公式)
  create(root) {           // 把自己的 UI 與邏輯全建進 root,狀態關在自己的閉包裡
    // ...
    return { destroy() {} }; // 選用:切換離開時收尾(例如停掉動畫)
  },
});
```

切換圖時,`App` 會先呼叫上一個圖的 `destroy()`(若有),再把顯示區清空、重建新的圖,所以圖與圖之間不可能互相污染。

**新增一個圖** = 多寫一個 `registerGraph({...})`,它會自動出現在底部按鈕列。
**移除一個圖** = 刪掉那一段。核心完全不動。

### 運算式解析器

圖 3 自寫了一個小型解析器(`tokenize → shunting-yard → RPN → eval`),支援:

- 運算子 `+ - * / ^`、括號、運算優先序、一元負號。
- **隱含乘法**:`5x` → `5*x`、`2sin(x)` → `2*sin(x)`、`(x-1)(x-2)`。
- 函數 `sin cos tan sqrt abs exp ln log`、常數 `pi e`。
- 找根用數值法:在範圍內密集取樣,偵測符號改變的區間,再以二分法逼近,最後去重、四捨五入到漂亮數字。

## 安全性

- 圖 3 的運算式**不使用 `eval()`**:自寫解析只做數學求值,不執行任何使用者輸入的程式碼。
- 輸入無法解析時,該行顯示友善錯誤訊息,而非默默算出錯誤答案。

---

## 與 calcpad 的架構差異(刻意的取捨)

- `calcpad` 用 ES modules + 多檔,需本機 HTTP server 才能開。
- `curvelab` 刻意做成**單一檔 + 傳統 `<script>`**,好處是**雙擊就能離線開**、好搬移;代價是所有程式集中在一個檔案。日後檔案變大時,可比照 calcpad 拆成多檔(把各圖模組、繪圖工具獨立),核心邏輯不變。

---

## 已知取捨

- 找根範圍預設 `-30 ~ 30`、步進 `0.02`;範圍外的根,或彼此非常靠近的根,可能漏抓。
- 各圖的可視範圍固定;使用者輸入超出範圍的函數會被裁切(只畫得下的部分)。
- 圓的 `x`、`y` **不開放滑桿調**,因為它們是「點的座標」而非參數;只有半徑 `r` 可調。這是刻意的教學設計,避免把座標和參數混為一談。
- 配色用 CSS 變數 + `prefers-color-scheme`,自動跟隨系統深淺色;SVG 顏色於載入時從 CSS 變數讀出實際色碼。

## 本機開發

單一檔,**直接雙擊 `index.html` 用瀏覽器開即可**(不需要 server,因為沒有用 ES modules、也沒有 `fetch` 本機檔案)。

若想用 server:

```bash
cd curvelab
python3 -m http.server 8010
# 開 http://127.0.0.1:8010/
```

本 repo 用 GitHub Pages 發佈。app 全用相對路徑、無建置步驟,放在子路徑(如 `/curvelab/`)即可運作。
