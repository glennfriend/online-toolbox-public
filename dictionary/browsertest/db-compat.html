這台瀏覽器的環境

sql.js (SQLite/WASM) 測試中…

純 WASM + JS,無特殊需求 → 相容性最廣。預設整個 DB 載入記憶體才能查。

DuckDB-Wasm 測試中…

用 Web Worker + WASM;高效版需 SharedArrayBuffer(要 COOP/COEP,GitHub Pages 設不了)→ 會自動退非多執行緒版,WASM 較大、首次載入較久。

官方 SQLite-Wasm + OPFS 測試中…

DB 放在瀏覽器 OPFS、按頁讀取(不整包載入記憶體)。重新整理本頁,「累計開啟次數」會 +1 → 代表持久化成功(這就是「第二次免下載、超快」的快取)。用 SAHPool VFS,不需 COOP/COEP。
