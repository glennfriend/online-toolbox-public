# Scrollshot

輸入網址,產生**整頁(含卷動)**的截圖,可下載 / 複製。純前端、單頁應用;截圖本身由第三方服務產生。

線上版:<https://glennfriend.github.io/online-toolbox-public/scrollshot/>

---

## 功能目的

把「我想要某個網頁的完整長截圖」變成貼上網址就拿到圖。重點是**整頁(含卷動)**,不是只截可見區。

- 輸入網址 → 取得整頁截圖並顯示。
- **下載**成 PNG、或**複製**圖片(可直接貼到 Word / Teams / Trello)。
- 可切換「整頁 / 只截可見區」。
- 跑流程時**逐步顯示訊息**(解析網址 → 嘗試服務 → 產生圖片);某步失敗標 ✗,滑鼠移上看原因。
- **失效自動切換**:截圖服務有時會不能用,所以會依序嘗試多家(見下),第一個能產出可載入圖片的就用,並**記住「上次成功的服務」**(localStorage)下次優先用;失敗就自動換下一家。

### 截圖服務與失效切換

依序嘗試(上次成功的會排到最前面):

| 服務 | 整頁 | 下載/複製 | 備註 |
|---|---|---|---|
| **Microlink** | ✓ | ✓(有 CORS) | 首選 |
| **thum.io** | ✓ | ✗(無 CORS,僅顯示/原圖另存) | 備援 |
| **mShots**(WordPress.com) | ✗ 只截可見區縮圖 | ✓(有 CORS) | 最後備援;首次對新網址可能先回「產生中」的圖 |

每次以 `Image()` 確認回傳的圖真的載得出來才算成功,否則換下一家。

> 行為提醒:因為「記住上次成功的服務」,一旦某次切到備援(例如 Microlink 暫時掛掉),之後會**持續優先用該備援**,直到它也失敗才會換回。

### 只收信譽明確的服務(為什麼)

只用 microlink、thum.io、Automattic 的 mShots,不接來歷不明的服務。風險在於:**目標網址與其畫面會送給該服務**(隱私);它可能回傳**錯誤或被竄改的圖**;若服務要求**載入它的 JS**,那段程式會在你的頁面執行(等於 XSS / 可被植入)——本工具只 `fetch` 圖片/JSON、從不載入第三方 JS,已避開這類執行風險,但隱私與「圖片可信度」仍取決於服務本身,故只用信譽明確者。

### 為什麼需要第三方服務(重要)

純前端(瀏覽器)**無法**對任意外部網址截圖:同源政策禁止讀別站像素,很多站也用 `X-Frame-Options`/CSP 拒絕被嵌入。所以真正「把網頁跑出來拍照」一定要有伺服器端的無頭瀏覽器。本工具的做法是讓**前端維持純靜態**,只負責「組出服務網址、顯示/下載/複製回傳的圖」,截圖交給免金鑰的第三方服務。

代價:有免費額度限制、可能有浮水印,且**目標網址會送給第三方**(勿截含機密的內網頁面)。

## 結構

純前端 ES modules,自包含。核心(殼層)與「截圖服務」分離,服務可插拔。

```
scrollshot/
├── index.html            版面 + 載入 js/main.js
├── styles.css
└── js/
    ├── main.js           殼層:流程、失效自動切換、記住上次成功的服務、下載/複製、步驟訊息
    ├── services/         每個截圖服務一個模組(可插拔;陣列順序 = 預設嘗試順序)
    │   ├── index.js      服務登記表 SERVICES
    │   ├── microlink.js  Microlink:免金鑰、有 CORS → 可下載/複製(首選)
    │   ├── thumio.js     thum.io:免金鑰、網址即圖;多半無 CORS → 以「顯示」為主
    │   └── mshots.js     mShots(WordPress.com):免金鑰、有 CORS、可下載;只截縮圖、非整頁
    └── lib/              與服務無關的通用模組
        ├── dom.js        el()
        ├── download.js   把 Blob 下載成檔
        └── clipboard.js  把影像 Blob 複製成 image/png
```

### 服務介面(可插拔的關鍵)

`main.js` 不認識任何特定服務,只透過統一介面呼叫。每個服務模組回傳:

```js
{
  id: 'microlink',
  name: 'Microlink(可下載 / 複製)',
  // capture(url, { fullPage }) -> { imageUrl, meta?, getBlob? }
  //   imageUrl : 直接給 <img> 顯示的網址
  //   meta     : 可選,{ width, height }
  //   getBlob(): 可選,取得可下載/複製的影像 Blob;沒有或失敗 → 退回「顯示 + 原圖另存」
  async capture(url, opts) { /* ... */ },
}
```

**新增一個服務** = 寫一個 `services/xxx.js`,在 `services/index.js` 的 `SERVICES` 加一筆;核心不用動。
**下載/複製的關鍵在 CORS**:服務回傳的圖能被 `fetch` 成 blob(有 CORS)才能下載/複製;否則自動降級成「顯示 + 用『原圖』連結另存」。

## 已知限制

- **抓的是網頁「初次渲染」的完整高度**:一般頁面(含一定程度的 lazy 內容)能完整抓到;連 YouTube 頻道頁這種,實測也能抓到約 7300px(好幾排影片)。
- **無限捲動 / 捲動才載入的頁面抓不全**:免金鑰服務無法「主動往下捲動載入更多」(實測 microlink 的 `waitForTimeout`、`scroll` 參數都不會讓抓到的高度變高),所以只會拿到初次載入的那一段,不是整個無限長的內容。要「捲動載入更多再抓」得改用有 auto-scroll 選項的付費(需金鑰)服務 + 後端。
- 免費額度限制、可能浮水印;目標網址會送給第三方,勿截含機密的內網頁面。

## 本機開發

ES modules 需 HTTP server(不能 `file://` 開):

```bash
cd scrollshot && python3 -m http.server 8009   # 開 http://127.0.0.1:8009/
```
