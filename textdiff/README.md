# TextDiff

兩邊貼上文字,**嚴格**比對差異——重點是把一般比對工具會忽略的東西也抓出來:**看不見的字元(空白、Tab、零寬、NBSP、全形空格)、Unicode 同形字、NFC/NFD 正規化、行尾(CRLF/LF/CR)、BOM**。純前端、不上傳。

線上版:<https://glennfriend.github.io/online-toolbox-public/textdiff/>

---

## 功能目的

解決「**明明看起來一樣,系統卻說不一樣 / 比對工具看不出差別**」的痛點:

- **並排差異**:行級比對,改動的行再做字元級高亮(增/刪)。
- **顯示不可見字元**:零寬字元、NBSP、全形空格、控制字元、bidi 一律標成看得見並附碼點說明;一般空白可用開關顯示(空格 ·、Tab ⇥)。
- **嚴格報告**:總結論(完全相同 / 畫面相同但 NFC·NFD 不同 / 只差空白·不可見 / 內容不同)+ 條列(行尾不一致、BOM、各邊的隱形字元、同形字疑慮、尾端空白)。
- **同形字疑慮**:同一個詞混用拉丁與西里爾/希臘字母(肉眼相同的偽造字)會被標出(含 Trojan Source / bidi 風險)。
- **比較模式**(按鈕,各有預設範例):逐字嚴格(顯示全部行)/ 文章(只看差異,折疊相同的行)/ 程式碼 / **JSON 結構化**(正規化成相同 key 順序後並排比對,不受順序與排版影響)。
- **選項開關**(各模式自訂、各自記住狀態):顯示空白字元、搬移偵測、即時更新(不延遲)、忽略大小寫、忽略空白。

> 預設帶一組示範:兩段「畫面幾乎一樣」的文字,B 偷偷用了 NFD、NBSP、零寬空格、西里爾同形字。

## 結構

純前端 ES modules,自包含。殼層只管 DOM/事件,邏輯在純函式模組(可插拔、好測)。

```
textdiff/
├── index.html
├── styles.css
└── js/
    ├── main.js      殼層:兩輸入框、debounce、並排渲染、報告
    ├── diff.js      核心:LCS 行/字元比對(純函式)
    ├── reveal.js    把不可見/危險字元轉成看得見的 HTML
    ├── inspect.js   嚴格分析:行尾/BOM/隱形字元/正規化/同形字
    └── unicode.js   共用:不可見字元對照表與小工具
```

要加新檢查(例如更多可疑字元、結構化 JSON/CSV 比對)= 在 `inspect.js` 多一條,或新增模組;殼層不太需要動。

## 為什麼不做「直接編輯」模式(刻意的決定,不要再加)

> 曾經做過一個「直接在輸入框內、邊打字邊上色」的模式(backdrop 技法:textarea 文字設透明、只留游標,後面疊一層對齊的上色層)。試過之後**移除**,原因記在這裡,免得日後又忘記、又重做一次:

1. **它不利於「看差異」**——而這正是本工具的目的。直接編輯的兩個輸入框各自顯示自己的文件,**兩邊不會逐行對齊**:A 刪一行,B 不會補空行,之後整段錯位,反而看不出差異。要看差異,**並排面板**(左右對齊、行號、折疊相同行、字元級紅綠)才是對的呈現。
2. **backdrop 有先天限制**:為了讓上色層跟透明文字「逐字對齊」,**不能用任何會改變字寬的標記**(空白點 `·`、Tab `⇥`、不可見字元符號、移出/移入標籤)——於是反而看不到 textdiff 最核心的「不可見字元符號」。
3. **透明文字 + 防抖**會造成「連續打字時整段字消失、停下才浮現」;要再為它特例化即時渲染、捲動同步等,複雜度不值得。

**結論**:直接編輯是「**編輯體驗**」,不是「**比對體驗**」。本工具專注 diff,一律用**並排面板**呈現;需要邊改邊看,用一般並排模式邊改邊比即可(它本來就會即時/防抖更新)。

## 已知取捨 / 之後可加

- 完整 Unicode 字元名稱、官方 confusables 對照表(更精準的同形字判定)目前用 `\p{Script=…}` 啟發式;要更準可內嵌資料集。
- 結構化比對(JSON 依 key、CSV 依格)規劃為第二階段。
- 大輸入有上限(預設前 5000 行,超過會出橫幅提示)。
